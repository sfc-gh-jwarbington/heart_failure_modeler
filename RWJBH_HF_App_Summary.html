What Is This?

We have built an interactive clinical decision-support application that guides the development of a multimodal heart failure 30-day readmission risk prediction model for RWJBarnabas Health's 12 acute care hospitals across New Jersey.

The application is a single, self-contained HTML tool that serves as the command center for the entire model lifecycle — from cardiologist-led factor review through Snowflake-based training and deployment to Epic Hyperspace integration.

What Problem Does It Solve?

Current HF readmission models (LACE, HOSPITAL score) rely on structured EHR data alone and plateau at AUC 0.59–0.65. Our approach combines seven data modalities to push performance toward AUC 0.75–0.82:

Data CategoryExample FactorsImpact
Structured ClinicalEjection fraction, BNP, eGFR, hemoglobin, sodiumBaseline
NLP / UnstructuredDischarge summary sentiment, med adherence mentions, functional status in nursing notes+5–9% AUC
Social DeterminantsADI, food insecurity, housing instability, insurance gaps+2–4% AUC
BehavioralAppointment no-shows, smoking status, substance use, depression (PHQ-9)+1–3% AUC
Imaging / DiagnosticsEcho reports (AI-parsed), chest X-ray congestion, ECG rhythm classification+1–2% AUC
Temporal / TrajectoryWeight trend (7-day), BNP trajectory, prior HF admissions (12-month)+2–4% AUC
Post-Discharge7-day follow-up, RPM/telehealth compliance, discharge med reconciliation+1–3% AUC
Why this matters financially: RWJBH hospitals face CMS HRRP penalties for excess readmissions. Even a 1–2% improvement in readmission rates represents significant savings. In FY2026, 240 hospitals nationally face penalties ≥1%. FY2027 adds Medicare Advantage data to all 6 measures, affecting 75–82% of hospitals.

What's Inside the Application

The app is organized into seven navigable sections:

1. System Overview

Hospital cards for all 12 RWJBH facilities with CMS star ratings and current HF readmission rates. Includes the evidence case for multimodal modeling (AUC comparison across structured-only, +NLP, and full multimodal approaches).

2. Quality Benchmarks

Comprehensive benchmarking with interactive charts:

  • Readmission rates: RWJBH hospitals vs. national leaders (Cleveland Clinic, NYU, Mt. Sinai) and CMS national average
  • NJ state context: Highest/lowest readmission states, NJ's position (~14.7%), HRRP penalty landscape
  • National trend timeline: Medicare readmission rates 2010–2025 with HRRP, COVID, and MA milestones
  • Mortality benchmarks: RWJBH hospital-level 30-day risk-standardized mortality rates (NQF #0229), comparison to Mayo/MGH/Cleveland Clinic, in-hospital and age-adjusted mortality trends 2000–2024, NJ mortality context

3. Research & Evidence Base

Seven curated research tabs with 50+ citations across:

  • RWJBH-specific publications
  • Foundational evidence (Frizzell, Rajkomar, Mortazavi)
  • NLP / unstructured data evidence
  • Social determinants of health evidence
  • Emerging factors (PubMed 2023–2025)
  • Code sets & data standards (ICD-10, SNOMED CT, LOINC, RxNorm, CPT)
  • State & policy evidence (Vizient, HRRP analysis, regional variation studies)

4. RWJBH Heart Stories

Curated patient stories and heart care content from across the RWJBH system, providing clinical context and human perspective on the readmission challenge.

5. State Case Studies

Six state-level case studies examining why readmission rates vary geographically: Massachusetts (highest at 15.3%), NJ/RWJBH (~14.7%), Idaho (lowest at 13.3%), West Virginia, Nevada, and synthesized lessons for RWJBH.

6. Risk Factor Review (Interactive)

The core clinical workflow. Cardiologists use this interface to review all 91 candidate risk factors:

  • Include/exclude toggle for each factor
  • Drag-and-drop reordering to prioritize factors
  • Editable strength ratings (High / Moderate / Low / TBD)
  • Evidence hyperlinks to PubMed/CMS sources for each factor
  • Category filtering (Structured, NLP, SDoH, Behavioral, Imaging, Temporal, Post-Discharge, External)
  • Custom factor addition for clinician-nominated factors not in the initial list
  • Data type, data source, and key code set annotations for every factor

7. Review & Model Creation

Phase tracking dashboard with review status controls, CSV/summary export, and a 5-phase development roadmap (see below). Status changes are timestamped with a persistent audit log.

The 5-Phase Development Roadmap

1
Cardiologist Review of Risk Factors (current phase)
Clinical review of 91 candidate factors. Inclusion/exclusion decisions documented with strength ratings, priority order, and code set mappings. Deliverable: approved risk factor list (CSV export).
2
Snowflake Data Element Scan & Gap Analysis
Map each approved factor to available data in RWJBH's Snowflake warehouse (Epic Clarity/Caboodle, claims, ADT). Identify structured availability, NLP extraction needs, and external data enrichment requirements. Deliverable: data availability matrix with coverage % and gap remediation plan.
3
Model Create, Train & Test Cycles
Iterative development in Snowflake using Snowpark ML and Snowflake Model Registry. Gradient boosting, logistic regression, and neural network candidates. k-fold cross-validation, hyperparameter tuning, fairness/bias evaluation. Target: AUC > 0.75.
4
Model Deployment for Inference
Real-time inference at two trigger points: index admission and index discharge. Snowpark Container Services or warehouse-based serving. Scoring pipeline writes to risk score table. Deliverable: production pipeline with monitoring and drift detection.
5
Epic Hyperspace Integration
Risk scores surface in Epic patient charts via Snowflake–Epic integration. BPA alerts for high-risk patients, discharge planning workflows with model-driven recommendations. CDS Hooks and/or Snowflake External Functions for real-time delivery.

How This Enables Other Models

While this application targets HF readmission, the framework is reusable across any clinical risk prediction use case:

  • Sepsis early warning — same multimodal approach (vitals trajectory + NLP + labs)
  • Surgical complication prediction — imaging + structured + temporal patterns
  • ED return visits / observation stays — behavioral + SDoH + post-discharge signals
  • Mortality risk stratification — already benchmarked in the app
  • Length-of-stay prediction — same factor categories, different outcome variable
  • CHF progression modeling — longitudinal EF trajectory + BNP trends + med adherence

The Snowflake platform provides the foundation: centralized data warehouse, Snowpark ML for training, Model Registry for versioning, Container Services for inference, and native connectors to Epic. Once the first model is deployed, subsequent models follow the same pipeline with different feature sets and outcome definitions.

Technical stack: Snowflake (data warehouse + ML platform) • Snowpark ML (model training) • Snowflake Model Registry (versioning) • Snowpark Container Services (inference) • Epic Clarity/Caboodle (source data) • Epic Hyperspace + CDS Hooks (clinical integration)

How to Use the Application

  1. Open the HTML file in any modern browser — no installation, server, or credentials required
  2. Navigate using the top nav bar across all seven sections
  3. Review benchmarks to understand where RWJBH stands relative to national leaders
  4. Explore the evidence base across seven research tabs and state case studies
  5. Review risk factors — toggle inclusion, reorder by priority, adjust strength ratings
  6. Add custom factors if the clinical team identifies factors not in the initial 91
  7. Mark review as complete when all factor decisions are finalized (timestamped audit log)
  8. Export the approved factor list as CSV or formatted summary for downstream use
State is preserved. Review status and audit log are saved in your browser's local storage. Closing and reopening the file will restore your progress. To reset, clear your browser's local storage for this file.
